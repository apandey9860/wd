HTTP and Security:

1. What is the Hypertext Transfer Protocol (HTTP)?

HTTP is a set of rules working at application layer of TCP/IP protocol that allows transmission of documents (e.g. HTML) between client and web server.

2. Why is HTTP called a stateless protocol?

HTTP is a stateless protocol, means that server doesn't retain any information about its client's requests and connection between them exists until transaction is being conducted.

3. Describe the three phases of establishing a session using client-server protocols such as HTTP.

Three phases of HTTP sessions consist:

(i) Establishing a connection

First, the client establishes the connection to transport layer (TCP) The default port for HTTP servers on a computer is 80.

(ii) Sending a client request

After the connection is established, client sends a request consisting of text directives, separated by CRLF (Carriage return, Line Feed). Division of CRLF contains request method followed by its parameters, HTTP header etc.

(iii) Response of Server

The server acknowledges the request and replies with a status code and other appropriate datas like HTTP headers. Status code is 200 for successful request and 404 for not found.

5. What are the HTTP response status code and What do the HTTP status codes, 200, and 404 indicate?

HTTP response status codes indicate if a specific HTTP request has been successfully completed. If HTTP status code is 200, it means that request was successful and 404 means that server cannot find the request resource.

6. What are HTTP cookies? Give an example.

An HTTP cookie is a piece of data that a server sends to the user's web browser that is use to confirm if two request came from the same browser.
Session cookies is an example of cookies.

7. Why are cookies a security issue?

Since cookies are used to identify a user and their authenticated session in web application, stealing of cookie can lead to hijacking the authenticated user's session.

8. Describe session hijacking and XSS

Session is a certain time frame dedicated for communication between two devices, two systems or two part of systems. When a session is hijacked, the attacker uses an exploit on a device and a host to disconnect that device from communication and attacker fakes its identity as the original device to receive private information from server.

Cross-site scripting or XSS are injection attacks in which malicious scripts are injected in trusted websites. The malicious script can access any cookies, session tokesns or other sensitive information retained in the browser.

9. Describe Cross-site request forgery(CSRF)

Cross-Site Request Forgery (CSRF) is an attack that forces an end user to execute unwanted actions on a web application in which they're currently authenticated mainly through email, chat or comments forum. CSRF attacks specifically target state-changing requests, not theft of data, since the attacker has no way to see the response to the forged request.

10. What are Cross-site HTTP requests (CORS) and what requests use CORS?

Cross-Origin Resource Sharing (CORS) is a mechanism that uses additional HTTP headers to tell a browser to let a web application running at one origin (domain) have permission to access selected resources from a server at a different origin.

Invocations of XMLHttpRequest, Web Fonts, Images/video frames drawn to canvas uses CORS.
